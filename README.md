# HandFlow

Система за разпознаване на жестове с ръка чрез Time-of-Flight сензори. Позволява безконтактно управление на smart home устройства — включване/изключване, сила на звука, заглушаване и др.

## Хардуер

| Компонент           | Описание                                     |
| ------------------- | -------------------------------------------- |
| STM32F401 Nucleo-64 | Основна платка (микроконтролер)              |
| X-Nucleo-53L0A1     | Разширителна платка с 3× VL53L0X ToF сензора |
| ESP-01              | WiFi модул за мрежова комуникация            |

ESP-01 е свързано чрез UART на пинове 3v3 - 16; GND - 20; en - PA8(23);
TX - PA10(33); Rx - PA9(21);

Сензорите са наредени физически: `[LEFT] [CENTER] [RIGHT]`

## Библиотеки

Инсталират се от Arduino Library Manager:

- **X_NUCLEO_53L0A1** от STMicroelectronics — включва:
  - `stmpe1600_class` (GPIO expander)
  - `vl53l0x_x_nucleo_53l0a1_class` (ToF сензори)
  - `tof_gestures` / `tof_gestures_DIRSWIPE_1` (ST gesture library)

## Разпознавани жестове

| Жест            | Символ | Описание                                  |
| --------------- | ------ | ----------------------------------------- |
| `SWIPE_RIGHT`   | ←→     | Ляво → дясно                              |
| `SWIPE_LEFT`    | →←     | Дясно → ляво                              |
| `SWIPE_DIAG_UR` | ↗      | Долу-ляво → горе-дясно                    |
| `SWIPE_DIAG_DR` | ↘      | Горе-ляво → долу-дясно                    |
| `SWIPE_DIAG_UL` | ↖      | Долу-дясно → горе-ляво                    |
| `SWIPE_DIAG_DL` | ↙      | Горе-дясно → долу-ляво                    |
| `VERTICAL_UP`   | ↑      | Ръката се вдига (разстоянието расте)      |
| `VERTICAL_DOWN` | ↓      | Ръката се снишава (разстоянието намалява) |

**Диагонална логика:** определя се от минималното разстояние на LEFT и RIGHT сензорите по цялото движение. По-малко разстояние = ръката е ниско/близо; по-голямо = ръката е нагоре/далеч.

**Вертикал:** засича се когато swipe прозорецът изтече, но ръката е все още над сензорите.

## State machine

```
IDLE → SWIPE_DETECT → COOLDOWN → IDLE
                    ↘
              VERTICAL_CHECK → COOLDOWN → IDLE
```

| Състояние        | Описание                                                         |
| ---------------- | ---------------------------------------------------------------- |
| `IDLE`           | Чака поява на ръка (debounce: 3 кадъра)                          |
| `SWIPE_DETECT`   | ST библиотеката следи за horizontal swipe (прозорец 1000 ms)     |
| `VERTICAL_CHECK` | Ако swipe не е засечен, следи вертикалното движение (до 2000 ms) |
| `COOLDOWN`       | Изчаква ръката да напусне, после изпълнява action-а              |

## Поддържани действия (Sony TV)

Проектът в момента управлява Sony TV чрез Sony REST API и Wake-on-LAN:

| Жест                                 | Действие               |
| ------------------------------------ | ---------------------- |
| `SWIPE_RIGHT` (без активен контекст) | Влиза в режим TV       |
| `SWIPE_RIGHT` (TV режим)             | TV ON                  |
| `SWIPE_LEFT` (TV режим)              | TV OFF                 |
| `SWIPE_DIAG_UR` (TV режим)           | Mute                   |
| `SWIPE_DIAG_DR` (TV режим)           | Unmute                 |
| `VERTICAL_UP` (TV режим)             | Volume +5              |
| `VERTICAL_DOWN` (TV режим)           | Volume -5              |
| `SWIPE_DIAG_DL`                      | Изход от всички режими |

### Ключови параметри

| Параметър               | Стойност | Описание                                      |
| ----------------------- | -------- | --------------------------------------------- |
| `HAND_PRESENT_MM`       | 400 mm   | Под това разстояние = ръка пред сензора       |
| `DIAG_THRESHOLD_MM`     | 40 mm    | Мин. разлика minL/minR за диагонален swipe    |
| `SWIPE_TIMEOUT_MS`      | 1200 ms  | Прозорец за разпознаване на swipe             |
| `VERTICAL_TIMEOUT_MS`   | 2000 ms  | Макс. престой в VERTICAL_CHECK                |
| `VERTICAL_THRESHOLD_MM` | 30 mm    | Мин. промяна на разстоянието за вертикал      |
| `HAND_DETECT_FRAMES`    | 3        | Последователни кадъра за потвърждение на ръка |

## Как работи

1. Трите VL53L0X сензора се инициализират на различни I²C адреси (`0x12`, `0x14`, `0x16`) чрез STMPE1600 GPIO expander.
2. В основния цикъл всички сензори се четат паралелно.
3. Данните се подават непрекъснато към ST gesture библиотеката.
4. State machine-ът класифицира движението и изпраща съответната команда към TV-то чрез ESP-01.
